# دليل اختبار رحلة التاجر في Kaleem Stores

## 1. هدف الاختبار

اختبار رحلة التاجر في Kaleem Stores من التسجيل حتى استقبال ومعالجة طلب.

يهدف هذا الاختبار إلى التأكد من أن التاجر يستطيع تنفيذ الرحلة الكاملة بدون مساعدة خارجية، وأن تجربة المستخدم UX مناسبة للإطلاق.

---

## 2. ما لا يشمله الاختبار

- لوحة الأدمن العامة للمنصة.
- إعدادات السيرفرات أو Cloudflare.
- مزودي الدفع الخارجيين.
- مزودي الشحن الخارجيين.
- إدارة القوالب من منصة الإدارة العامة.
- إدارة خطط الاشتراك من الأدمن العام.

## 4. طريقة تنفيذ الاختبار

1. **كل مختبر يختار وثيقة واحدة** فقط تناسب نوع المتجر المراد اختباره.
2. **يستخدم بريد إلكتروني جديد** أو قابل للاستخدام للاختبار.
3. **يستخدم slug فريد** بالصيغة: `[slug]-[اسم المختبر]-[MMDD]`
   - مثال: `tech-zone-yemen-ahmed-0517`
   - لا تستخدم slug موجود مسبقًا. إذا ظهر أن الرابط مستخدم، أضف رقمًا أو اسمك في النهاية.
4. **يوثق كل مشكلة** بصورة أو فيديو قصير إن أمكن.
5. **يرفع الصور/الفيديوهات** مع ملف الملاحظات.
6. **يسجل وقت بداية ونهاية الاختبار** في جدول معلومات المختبر.
7. **يملأ نموذج تسجيل المشكلة** لكل مشكلة يكتشفها.
8. **يملأ تقييم تجربة التاجر** في نهاية الوثيقة.

## 5. معايير نجاح الاختبار

يعتبر السيناريو ناجحًا إذا استطاع المختبر:

- [ ] إنشاء حساب تاجر.
- [ ] إنشاء متجر.
- [ ] إكمال الإعدادات الأساسية (هوية، رابط، شعار، سياسات).
- [ ] إنشاء تصنيفات وبراندات وخصائص.
- [ ] إعداد الفلاتر من لوحة التاجر والتأكد من ظهورها في المتجر.
- [ ] إنشاء منتجات بسيطة ومتغيرة.
- [ ] إعداد الدفع (طرق دفع يدوية).
- [ ] إعداد التوصيل أو الاستلام من المتجر.
- [ ] فتح المتجر كعميل.
- [ ] إتمام طلب.
- [ ] ظهور الطلب في لوحة التاجر.
- [ ] تغيير حالة الطلب حتى الاكتمال.
- [ ] تسجيل مشاكل UX بشكل واضح.
- [ ] ملء تقييم تجربة التاجر.

---

## 6. مخرجات المختبر المطلوبة

بعد إكمال الاختبار، يجب أن يسلّم المختبر:

1. **ملف الملاحظات**: الوثيقة المعدّلة مع تعبئة جميع الحقول الفارغة.
2. **صور أو فيديوهات**: لكل مشكلة تم اكتشافها.
3. **قائمة المشاكل**: جدول المشاكل مع تصنيف الخطورة.
4. **تقييم UX**: قسم تقييم تجربة التاجر في نهاية الوثيقة.
5. **قرار المختبر**:

   - [ ] **جاهز**: المتجر يعمل بشكل كامل وجاهز للإطلاق.
   - [ ] **جاهز بعد تعديلات بسيطة**: مشاكل منخفضة أو متوسطة فقط.
   - [ ] **يحتاج تحسينات قبل الإطلاق**: مشاكل عالية تحتاج حل.
   - [ ] **غير جاهز**: مشاكل حرجة تمنع الإطلاق.

---

## 7. ملاحظات عامة

- جميع الوثائق باللغة العربية ومصممة لمختبر غير تقني.
- لا تطلب أي وثيقة من المختبر دخول لوحة الأدمن العامة.
- الفوترة والاشتراك: إذا لم تكن مفعلة في بيئة الاختبار، يسجل المختبر ذلك كملاحظة ولا يوقف الاختبار.
- الكوبونات: ليست شرطًا لإكمال الطلب الأساسي.
- القوالب: كل وثيقة تحدد القالب الأساسي المناسب، ولا تجعل قوالب أخرى شرط نجاح.

## 8. التواصل

في حالة وجود أي استفسار أو مشكلة تقنية أثناء الاختبار، تواصل مع مدير المشروع.
