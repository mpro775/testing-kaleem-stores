5. استخدم فحص توفر الرابط إن ظهر.
6. اختر نوع النشاط:
   - `abayas` إذا كان المتجر يركز على العبايات.
   - `fashion` إذا كان يبيع أزياء نسائية متنوعة.
7. اختر العملة الافتراضية: YER.
8. اختر المنطقة الزمنية: Asia/Aden.

12. أدخل الدولة والمحافظة والعنوان التفصيلي.
13. افتح الخريطة وحدد موقعاً في صنعاء أو أمانة العاصمة.
14. اضبط ساعات العمل. مثال:

16. اكتب السياسات:

    - سياسة الاستبدال والاسترجاع، مع توضيح شروط الملابس: المنتج غير مستخدم، بنفس التغليف، خلال مدة محددة.

    - الشروط والأحكام.

## 22. قواعد عامة

لكل منتج، اختبر الحقول التالية عند توفرها:

- اسم عربي وإنجليزي.
- وصف عربي وإنجليزي.
- تصنيف واحد أو أكثر.
- Brand.
- نوع المنتج: single أو bundled أو digital إن كان مطلوباً.
- الحالة والظهور.
- السعر.
- سعر المقارنة `compareAtPrice` إذا كان هناك خصم ظاهر.
- خصم مدمج `inlineDiscount` إذا كان متاحاً، بنسبة أو مبلغ ثابت مع تاريخ بداية ونهاية.
- تكلفة المنتج `costPrice`.
- المخزون أو مخزون غير محدود.
- حد المخزون المنخفض.
- SKU و barcode.
- وزن وأبعاد.
- حد أدنى وأقصى للطلب.
- وسوم Tags مثل: عباية، فستان، سهرة، يومي، فخم.
- تسمية منتج Product Label مثل: جديد، الأكثر طلباً، محدود.
- رابط YouTube لفيديو عرض أو تنسيق إن وجد.
- SEO title و SEO description.
- منتجات مرتبطة.
- أسئلة المنتج مفعلة إن ظهر الخيار.
- الضرائب `isTaxable` و `taxRate` إن ظهرت.

# المرحلة العاشرة: تجربة العميلة في المتجر

3. افتح صفحة المنتجات.
4. اختبر البحث عن "عباية"، "فستان"، "حقيبة"، "طرحة".
5. اختبر التصنيفات.
6. اختبر الفلاتر حسب السمات إن ظهرت.
7. افتح صفحة عباية بمتغيرات.
8. اختبر تغيير اللون والطول.
9. افتح صفحة الفستان واختبر المقاس واللون.
10. افتح صفحة الحقيبة واختبر المقاس واللون.
11. افتح المنتج النافد.
12. أضف منتجاً للمفضلة.
13. أضف سؤالاً على منتج مثل: "هل الطول 56 مناسب لطول 165 سم؟"
14. أضف مراجعة إن كان متاحاً.

# المرحلة الرابعة عشرة: تغيير حالة الطلب

## 39. الحالات الفعلية

| الحالة التقنية | التسمية العربية |
|---|---|
| new | جديد |
| confirmed | مؤكد |
| preparing | قيد التجهيز |
| out_for_delivery | في الطريق |
| completed | مكتمل |
| cancelled | ملغي |
| returned | مسترجع |

## 40. الانتقالات المسموحة

| من | إلى |
|---|---|
| new | confirmed أو cancelled |
| confirmed | preparing أو cancelled |
| preparing | out_for_delivery أو cancelled |
| out_for_delivery | completed أو returned |
| completed | لا يوجد انتقال |
| cancelled | لا يوجد انتقال |
| returned | لا يوجد انتقال |

## 41. خطوات التنفيذ

1. غيّر الطلب من `new` إلى `confirmed`.
2. أضف ملاحظة إن ظهر حقل الملاحظة.
3. غيّره إلى `preparing`.
4. غيّره إلى `out_for_delivery`.
5. افتح صفحة تتبع الطلب وتأكد من ظهور الحالة.
6. غيّره إلى `completed`.
7. أنشئ طلباً ثانياً أو استخدم طلب اختبار آخر لتجربة الإلغاء.
8. جرّب انتقالاً غير مسموح مثل `new` إلى `completed` إن أمكن، وسجل هل تم منعه.

## 42. تحقق المرحلة

- [ ] الحالات المعروضة مطابقة للقائمة الفعلية.
- [ ] لا توجد حالة "قيد المراجعة" كحالة طلب.
- [ ] الانتقالات غير المسموحة ممنوعة أو غير معروضة.
- [ ] الملاحظات تحفظ إن وجدت.
- [ ] حالة العميلة في صفحة التتبع تتحدث.

# المرحلة الخامسة عشرة: الإشعارات

## 43. ما يجب فحصه

1. إشعار إنشاء طلب جديد.

6. تفضيلات الإشعارات إن كانت ظاهرة:
   - فورية.
   - ملخص يومي.
   - كتم.
7. التحديث الفوري إن وصل بدون تحديث الصفحة.

أحداث مهمة يجب تسجيلها إن ظهرت:

- `order.created`
- `inventory.low_stock`
- `inventory.out_of_stock`

## 44. تحقق المرحلة

- [ ] ظهر إشعار عند إنشاء الطلب.
- [ ] ظهر الطلب في مركز الإشعارات.
